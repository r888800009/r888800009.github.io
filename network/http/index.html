連線上的限制

客戶端若使用持續性連線，應該要透過伺服器限制同時連線， single-user的客戶端不應該同時超過2個連線到任何server或proxy。 RFC 2616

Idempotent

有哪些 method

  • GET, HEAD, PUT, DELETE

確保每次回傳結果都是相等的，不影響下次請求 1

意義

如果連線發生問題，冪等方法可以重新請求，並且預期是相同的
