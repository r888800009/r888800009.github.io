[網路安全] 公鑰與訊息認證

Fri, 2020 Mar 20

為何需要認證訊息

  • 確定訊息來自受信任的發送者: secret key
  • 訊息未被修改: 錯誤偵測碼
  • 訊息是否即時傳達，沒有受到延遲或重送: timestamp

確認內容來源

可能的方法

  • 對 message 加密
  • 對 message Hash，並且對hash加密
    • public key加密
    • Secret value: 運算效能不一定足夠使用public key，但依然需要安全保護的替代方法

Secure Hash function的性質

  • 可變化的輸入長度
  • 固定的輸出長度
  • 有效率的演算法
  • 單向: 無法透過hash還原
  • 弱碰撞: 任意x不可被碰撞，eg: A可以決定x, B嘗試碰撞出x’
  • 強碰撞: 攻擊者可以控制的x與x'也不可以碰撞, eg: B能夠控制x與x’
  • 偽隨機: 符合偽隨機測試標準
